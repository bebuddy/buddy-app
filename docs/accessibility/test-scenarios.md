# E2E 접근성 테스트 시나리오 (VoiceOver/TalkBack 중심)

## 시나리오 1: 화면 탐색 순서
- 대상: 홈/목록/채팅 목록 화면
- 단계
  1. VoiceOver/TalkBack 활성화
  2. 오른쪽 스와이프로 요소를 순차 탐색
  3. 주요 CTA(필터/탭/카드/하단 내비)까지 이동
- 기대 결과
  - 논리적 순서로 읽힘
  - 불필요한 장식 요소는 읽히지 않음

## 시나리오 2: 모달 포커스 트랩/복귀
- 대상: `AppModal` 사용 화면
- 단계
  1. 모달을 여는 버튼에 포커스
  2. 모달 열기
  3. Tab/스와이프로 모달 내 요소 탐색
  4. 닫기 실행
- 기대 결과
  - 모달 열림 시 첫 포커스가 제목/첫 입력으로 이동
  - 모달 밖으로 포커스가 나가지 않음
  - 닫힘 후 트리거 버튼으로 복귀

## 시나리오 3: 폼 오류 읽기/포커스 이동
- 대상: 로그인/회원가입/온보딩 폼
- 단계
  1. 필수 입력 누락 상태로 제출
  2. 오류 메시지 읽기
  3. 첫 오류 필드로 이동 확인
- 기대 결과
  - 오류 메시지가 `aria-live`로 읽힘
  - `aria-invalid` 설정된 필드로 포커스 이동

## 시나리오 4: 캐러셀 자동재생 제어
- 대상: `AppCarousel` 사용 화면
- 단계
  1. 자동재생 상태 확인
  2. 정지 버튼 클릭
  3. 이전/다음 버튼으로 이동
- 기대 결과
  - 현재 위치(예: 3/10)가 읽힘
  - 정지/재생 컨트롤이 명확히 읽힘

## 시나리오 5: 알림/토스트
- 대상: 저장/실패 등 알림 발생 플로우
- 단계
  1. 성공/실패 이벤트 발생
  2. 스크린리더 읽기 확인
- 기대 결과
  - `role="status"` 또는 `role="alert"`로 읽힘
  - 닫기 버튼이 접근 가능
